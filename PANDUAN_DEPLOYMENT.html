3

Jalankan script database

Ini adalah langkah yang paling penting. Klik menu SQL Editor di sidebar kiri:

Dashboard→ SQL Editor→ + New query
📋
Copy file database.sql Buka file database.sql dari ZIP yang kamu download. Pilih semua (Ctrl+A), copy (Ctrl+C), lalu paste ke SQL Editor Supabase. Klik tombol RUN ▶ di pojok kanan atas.
✅
Berhasil jika: Muncul pesan "Success. No rows returned" atau pesan sukses lainnya. Tidak ada teks merah/error.
❌
Jika muncul error merah: Coba jalankan ulang (klik Run lagi). Jika masih error, coba hapus semua isi SQL Editor dan paste ulang dari awal file.

⚡ Tambahkan Environment Variables — WAJIB!

Di halaman konfigurasi project (sebelum deploy), scroll ke bawah sampai bagian "Environment Variables". Tambahkan 2 variabel berikut:

# Variabel 1 NAME: NEXT_PUBLIC_SUPABASE_URL VALUE: https://xxxxxxxxxxxxxxxx.supabase.co ↑ URL yang kamu copy dari Supabase Settings → API # Variabel 2 NAME: NEXT_PUBLIC_SUPABASE_ANON_KEY VALUE: eyJhbGciOiJIUzI1NiIsInR5cCI6... ↑ anon public key dari Supabase Settings → API
🚫
Jangan lupa ini! Kalau tidak menambahkan env vars, aplikasi tidak bisa terkoneksi ke database dan akan error saat dibuka.

Troubleshooting

Solusi untuk masalah yang paling sering terjadi

❌ Login gagal — "Invalid login credentials"
Periksa: (1) Email dan password benar di Supabase Authentication → Users, (2) User sudah dikonfirmasi (tidak ada status "Unconfirmed"), (3) Supabase URL dan key sudah benar di Vercel env vars.
❌ Halaman kosong / error 500 saat deploy
Paling sering karena env vars belum diisi. Di Vercel: Settings → Environment Variables → pastikan kedua variabel ada. Setelah mengubah env vars, klik "Redeploy".
❌ Data tidak muncul / tampilan error "relation does not exist"
Script database.sql belum dijalankan atau ada error. Kembali ke Supabase SQL Editor dan jalankan ulang seluruh isi database.sql.
❌ Upload foto gagal
Cek di Supabase Storage → Buckets → pastikan bucket "lampiran" sudah ada. Jika belum ada, script mungkin tidak berjalan sempurna. Buat manual: Storage → New bucket → nama "lampiran" → Public.
❌ Login redirect ke halaman kosong (tidak masuk dashboard)
Redirect URL belum dikonfigurasi. Di Supabase → Authentication → URL Configuration → tambahkan: https://URL-VERCEL-KAMU.vercel.app/auth/callback
