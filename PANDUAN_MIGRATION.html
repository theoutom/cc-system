🔧 Panduan: Jalankan Migration SQL

Diperlukan agar fitur baru (asal organisasi, inventaris real-time, akses publik) berfungsi

⚠️ Kenapa fitur 2, 3, 5 belum tampil? Kolom asal_organisasi dan items_dipinjam belum ada di database Supabase kamu. Fiturnya sudah ada di kode, tinggal jalankan SQL di bawah ini dan semua langsung berfungsi.
1
Buka Supabase Dashboard
Login ke Supabase dan pilih project CC System kamu
https://supabase.com/dashboard

Klik project CC System → masuk ke dashboard project

2
Buka SQL Editor
Klik menu "SQL Editor" di sidebar kiri
🏠 Home 📊 Table Editor ⚡ SQL Editor ← klik ini 🔐 Authentication 📦 Storage
3
Buat Query Baru
Klik tombol "+ New query" di pojok kiri atas SQL Editor
+ New query

Akan muncul editor kosong siap diisi

4
Copy-paste isi file SQL
Buka file JALANKAN_DI_SUPABASE.sql dari folder yang kamu download, lalu copy semua isinya ke editor
-- CC SYSTEM — MIGRATION LENGKAP
-- Copy semua dari file JALANKAN_DI_SUPABASE.sql

ALTER TABLE peminjaman
  ADD COLUMN IF NOT EXISTS items_dipinjam UUID[];

ALTER TABLE peminjaman
  ADD COLUMN IF NOT EXISTS asal_organisasi TEXT;

... (lanjut sampai bawah)
💡 Tips: Ctrl+A untuk select all teks di editor Supabase sebelum paste, supaya tidak ada sisa query lama.
5
Klik Run / Jalankan
Tekan tombol hijau "Run" atau shortcut Ctrl+Enter
▶ Run

Tunggu beberapa detik sampai muncul output di panel bawah

6
Cek hasilnya di panel bawah
Pastikan output menampilkan "Migration berhasil!" dan angka kolom = 2
status               | kolom_baru_ada | total_policies
✅ Migration berhasil! |              2 |            11
❌ Kalau muncul error? Coba jalankan ulang — SQL ini aman dijalankan berkali-kali. Kalau masih error, screenshot pesan errornya dan hubungi pengembang.
7
Push kode ke GitHub → otomatis deploy ke Vercel
Semua file kode sudah siap, tinggal push dan tunggu deploy selesai
# Di terminal/command prompt, dari folder cc-system-fixed:

git add .
git commit -m "feat: akses publik, asal org, catatan alat, inventaris realtime"
git push origin master

Vercel akan otomatis deploy dalam ~1-2 menit setelah push

✓
Fitur yang langsung aktif setelah migration
Semua ini akan langsung berfungsi begitu migration dan deploy selesai
🏢
Kolom "Asal Organisasi/Pihak" tampil Muncul di form peminjaman (step 1) untuk jenis Organisasi & Eksternal. Tampil di list peminjaman sebagai badge abu-abu di sebelah nama kegiatan.
📝
Catatan alat tampil di grid Setiap card alat di grid Step 3 menampilkan catatan dari kolom catatan di tabel inventaris. Kalau kolom catatan kosong, tidak ada yang tampil.
🔴
Inventaris real-time dengan info peminjam Di halaman Admin → Inventaris, setiap alat yang dipinjam menampilkan nama kegiatan, asal organisasi, nama peminjam, dan estimasi kembali. Badge dibedakan: Dipinjam / Pending / Tersedia.
📱
Nomor telepon hanya menerima angka Input nomor telepon sekarang memfilter karakter non-angka secara otomatis dan memvalidasi minimal 8 digit.
🌐
Akses publik (tanpa login) Tombol "Pinjam Alat" dan "Kembalikan" di halaman login membuka form publik di /publik/pinjam dan /publik/kembali.

🎉 Selesai!

Setelah migration dijalankan dan kode di-push, semua fitur baru langsung aktif.
Tidak perlu restart server atau konfigurasi tambahan.
